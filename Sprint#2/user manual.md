📘 User Manual
ฟีเจอร์: รีวิวการเดินทาง (Passenger Review System)

User Story

As a passenger, I want to give a review for each ride that I took to support the community.

ในฐานะผู้โดยสาร ฉันต้องการให้รีวิวสำหรับแต่ละทริปที่ฉันใช้บริการ เพื่อสนับสนุนชุมชน

1️⃣ วัตถุประสงค์ของฟีเจอร์

     ฟีเจอร์นี้ช่วยให้ผู้โดยสารสามารถ:

         ให้คะแนนคนขับหลังจบทริป

         แสดงความคิดเห็นเกี่ยวกับการเดินทาง

         เลือกจุดเด่นของทริป (เช่น ตรงเวลา สะอาด ปลอดภัย)

         เลือกรีวิวแบบไม่แสดงชื่อ

         สนับสนุนและให้ข้อมูลแก่ผู้ใช้คนอื่น

2️⃣ เงื่อนไขการรีวิว

     สามารถรีวิวได้ ภายใน 7 วันหลังจากทริปเสร็จสิ้น

     1 ทริป รีวิวได้ 1 ครั้ง

     หากเกิน 7 วัน จะไม่สามารถรีวิวได้

3️⃣ ขั้นตอนการให้รีวิว

    Step 1: ไปที่เมนู “การเดินทางทั้งหมด”

         เข้าสู่ระบบในฐานะผู้โดยสาร
         เลือกเมนู การเดินทางทั้งหมด

     Step 2: กดปุ่ม “เขียนรีวิว”

         เลือกทริปที่ต้องการรีวิว
         กดปุ่ม เขียนรีวิว

         ระบบจะแสดงหน้าต่างเขียนรีวิว (Popup)

     Step 3: ให้คะแนนความพึงพอใจ

         เลือกจำนวนดาว (1–5 ดาว)

         ⭐ 1 ดาว = ไม่พึงพอใจมาก

         ⭐⭐⭐⭐⭐ 5 ดาว = พึงพอใจมากที่สุด

     Step 4: เลือกจุดเด่นของทริป

         สามารถเลือกได้มากกว่า 1 ข้อ เช่น:

         ตรงเวลา

         สะอาด

         มีมารยาท

         ขับปลอดภัย

         นั่งสบาย

     Step 5: เขียนความคิดเห็นเพิ่มเติม

         พิมพ์ความคิดเห็นในช่องข้อความ
         (แนะนำให้เขียนรายละเอียดเพื่อช่วยผู้ใช้คนอื่น)

     Step 6: เลือกแสดงชื่อหรือไม่

         หากต้องการรีวิวแบบไม่ระบุตัวตน
         ติ๊กเลือก:

         ☐ ซ่อนชื่อของฉัน (รีวิวแบบไม่ระบุตัวตน)

     Step 7: กด “ส่งรีวิวเลย”

         เมื่อกรอกครบแล้ว กดปุ่ม ส่งรีวิวเลย

         ระบบจะบันทึกข้อมูลเรียบร้อย

4️⃣ กรณีหมดเขตรีวิว

     หากเกิน 7 วัน
     ปุ่ม "เขียนรีวิว" จะถูกปิดการใช้งาน และแสดงข้อความว่า:

     เลยกำหนด 7 วันหลังการเขียนรีวิวแล้ว ไม่สามารถเขียนรีวิวทริปนี้ได้

5️⃣ การดูรีวิวของตนเอง

     ผู้โดยสารสามารถดูรีวิวที่ตนเองเคยเขียนได้ในหัวข้อ:

         รีวิวของคุณ

6️⃣ การแสดงผลรีวิวในระบบ

     หลังจากส่งรีวิวแล้ว รีวิวจะแสดงใน:

         หน้าโปรไฟล์คนขับ

         รีวิวล่าสุดจากชุมชน

7️⃣ การจัดการรีวิว (ฝั่ง Admin)

     Admin สามารถ:

         ดูจำนวนรีวิวทั้งหมด

         ตรวจสอบรีวิวคะแนนต่ำ (1–2 ดาว)

         ค้นหาตาม Driver ID

         ตรวจสอบความคิดเห็นที่ไม่เหมาะสม


📘 User Manual
ฟีเจอร์: รายงานเหตุการณ์ (Incident Reporting) สำหรับคนขับ

User Story:

As a driver, I want to report incidents to the admin and get the update on the filed case.

ในฐานะคนขับ ฉันต้องการรายงานเหตุการณ์ไปยังผู้ดูแลระบบ และติดตามสถานะของเคสที่แจ้งไป

1️⃣ วัตถุประสงค์ของฟีเจอร์

     ฟีเจอร์นี้ช่วยให้ คนขับ สามารถ:

           แจ้งเหตุฉุกเฉินหรือปัญหาระหว่างการเดินทาง

           แนบรูปภาพ/รายละเอียดเหตุการณ์

           ส่งตำแหน่งที่เกิดเหตุ

           ติดตามสถานะการดำเนินการของทีมงาน

           ดูผลสรุปเมื่อเคสถูกปิด

2️⃣ การแจ้งเหตุ (Report Incident)

     ขั้นตอนการใช้งาน

     Step 1: เข้าสู่ระบบ

         เข้าสู่ระบบในฐานะ คนขับ

     Step 2: กดปุ่ม “REPORT”

         ที่หน้าแรกของระบบ จะมีปุ่ม REPORT สีแดง มุมขวาล่าง

         กดเพื่อเริ่มแจ้งเหตุ

     Step 3: เลือกประเภทเหตุการณ์

     หน้าจอจะแสดงประเภทเหตุ เช่น:

         เหตุอุบัติเหตุ / เฉี่ยวชน

         เหตุเจ็บป่วย / ต้องการแพทย์

         ไม่ปลอดภัย / ถูกคุกคาม

         รถเสีย / ใช้งานต่อไม่ได้

         เหตุอื่น ๆ

         เลือกประเภทที่ตรงกับเหตุการณ์

     Step 4: กรอกรายละเอียดเหตุการณ์

     กรอกข้อมูล:

         รายละเอียดเหตุการณ์ (อธิบายสิ่งที่เกิดขึ้น)

         แนบไฟล์ภาพ/วิดีโอ (ถ้ามี)

         ระบบแนบตำแหน่ง (Lat/Lng) อัตโนมัติ

     Step 5: กด “ส่งรายงาน”

         เมื่อข้อมูลครบถ้วน กดปุ่ม ส่งรายงาน

         ระบบจะแสดงข้อความ:

         ส่งคำขอสำเร็จ
         ทีมงานได้รับการแจ้งเตือนแล้ว กรุณารอความช่วยเหลือ

3️⃣ การติดตามสถานะเคส

     หลังจากส่งรายงานแล้ว คนขับสามารถติดตามสถานะได้ที่:

         เมนู:

         โปรไฟล์ → ประวัติเหตุฉุกเฉิน (SOS)

         สถานะของเคส

         ระบบจะแสดงสถานะดังนี้:

         สถานะ	           ความหมาย
         ทีมงานตอบรับแล้ว	 แอดมินรับเรื่องแล้ว
         ปิดเหตุฉุกเฉินแล้ว	   เคสถูกแก้ไขเรียบร้อย
         ยกเลิก	            เคสถูกยกเลิก

4️⃣ รายละเอียดที่แสดงในแต่ละเคส

     เมื่อกดดูรายละเอียด จะเห็นข้อมูล:

         ข้อมูลคนขับ

         ตำแหน่งที่เกิดเหตุ (เปิดใน Google Maps ได้)

         รายละเอียดเหตุการณ์

         ไฟล์แนบ

         ลำดับเหตุการณ์ (Created / Responding / Resolved)

         หมายเหตุจากทีมงาน (เมื่อปิดเคส)

5️⃣ การทำงานของ Admin (เพื่อให้คนขับเข้าใจ Flow)

     เมื่อคนขับแจ้งเหตุ:

         เคสจะเข้าสู่ระบบ Admin

         แสดงในหน้า Incident Reports

         แอดมินสามารถ:

         กด “รับเรื่องแล้ว (Responding)”

         กด “ปิดเหตุฉุกเฉิน (Resolved)”

         ใส่หมายเหตุการดำเนินการ

         ระบบอัปเดตสถานะให้คนขับเห็นทันที

6️⃣ Dashboard ภาพรวมเคส (ฝั่ง Admin)

     แสดงจำนวน:

         Active

         Responding

         Resolved

         Cancelled

         Total

        เพื่อให้ทีมงานติดตามเคสเร่งด่วนได้

7️⃣ ข้อจำกัดของระบบ

     แนบไฟล์ได้สูงสุด 5 ไฟล์

     ขนาดไฟล์ไม่เกิน 20 MB ต่อไฟล์

     ต้อง Login ก่อนใช้งาน

     ระบบจะใช้ตำแหน่งปัจจุบันของผู้ใช้
